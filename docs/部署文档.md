# 连锁门店视频管理系统 — 部署文档

## 1. 系统要求

### 服务端
- **操作系统**: Ubuntu 22.04 LTS / CentOS 8+
- **Docker**: 24.0+
- **Docker Compose**: v2.0+
- **内存**: 最低 4GB，推荐 8GB
- **磁盘**: 最低 50GB（根据视频存储量调整）
- **网络**: 1Gbps 上行带宽（服务端）
- **端口**: 3000(API)、5173(管理后台)、3306(MySQL)、6379(Redis)、9000/9001(MinIO)、1883/8083/18083(EMQX)

### 客户端
- **操作系统**: Windows 10 64位 或更高
- **内存**: 最低 4GB
- **磁盘**: 根据视频数量，建议预留 50GB+
- **网络**: 需要互联网连接

## 2. 服务端部署

### 2.1 一键部署

```bash
# 克隆项目
git clone <repository-url> /opt/vdeio
cd /opt/vdeio/deploy

# 执行一键部署脚本
chmod +x install.sh
./install.sh
```

脚本会自动：
1. 检查 Docker 环境
2. 生成随机密钥并创建 `.env` 配置文件
3. 拉取 Docker 镜像并启动服务
4. 等待服务就绪并输出访问地址

### 2.2 手动部署

```bash
cd deploy

# 1. 创建配置文件
cp .env.example .env
vi .env  # 编辑配置

# 2. 启动所有服务
docker compose up -d

# 3. 查看服务状态
docker compose ps

# 4. 查看日志
docker compose logs -f server
```

## 4. 钉钉开放平台配置

### 4.1 创建应用

1. 登录 [钉钉开放平台](https://open-dev.dingtalk.com)
2. 进入「应用开发」→「网页应用」→「创建应用」
3. 填写应用名称：`门店视频播放系统`
4. 获取 **AppKey** 和 **AppSecret**

### 4.2 配置登录权限

1. 进入应用 → 「登录与分享」
2. 授权范围选择「全部员工」
3. 回调域名填写：`http://your-server:3000/api/v1/auth/dingtalk/callback`

### 4.3 填写配置

将获取的 AppKey 和 AppSecret 填入服务端 `.env`：

```
DINGTALK_APP_KEY=dingxxxxx
DINGTALK_APP_SECRET=xxxxx
```

重启服务：`docker compose restart server`

## 5. EMQX 配置

### 5.1 MySQL 认证

EMQX 已通过 Docker Compose 配置了 MySQL 认证后端，使用 `mqtt_user` 表进行设备认证。

### 5.2 Webhook 集成

在 EMQX Dashboard (http://your-server:18083) 配置 Webhook：

1. 进入「集成」→「Webhook」
2. 创建 Webhook：
   - URL: `http://server:3000/api/v1/webhooks/emqx`
   - 方法: POST
3. 添加触发器：
   - 事件: `client.connected` → 设备上线
   - 事件: `client.disconnected` → 设备离线

## 6. 客户端安装

### 6.1 Windows PC 安装

```powershell
# 方法一：使用安装脚本
.\deploy\client-install.ps1 -ServerUrl http://your-server:3000

# 方法二：手动安装
# 1. 复制客户端安装包到目标 PC
# 2. 运行 vdeio-client-setup.exe
# 3. 安装完成后首次启动
# 4. 使用钉钉扫码登录
```

### 6.2 首次登录

1. 启动客户端
2. 使用钉钉 APP 扫描屏幕上的二维码
3. 扫码成功后自动绑定当前门店
4. 系统自动同步视频并开始播放

## 7. 管理员操作手册

### 7.1 登录管理后台

1. 浏览器访问 `http://your-server:5173`
2. 使用管理员账号登录（默认: admin / 查看 .env）

### 7.2 上传视频

1. 进入「视频管理」
2. 点击「上传视频」
3. 拖拽或选择视频文件（支持 mp4、mov、avi）
4. 系统自动进行 AES-128 HLS 加密
5. 加密完成后状态变为「已完成」

### 7.3 创建推广活动

1. 进入「推广计划」
2. 点击「新建推广计划」
3. 填写标题、描述、时间范围
4. 选择已加密的视频
5. 选择目标门店
6. 点击「保存」→「发布」

### 7.4 监控设备

1. 进入「设备管理」
2. 查看设备在线/离线状态
3. 查看设备遥测数据（CPU、内存、磁盘）
4. 可下发远程命令：重启、同步、清空缓存

### 7.5 查看仪表盘

- 首页「仪表盘」显示：
  - 总视频数、活动数、在线设备数
  - 活动视频分布饼图
  - 设备在线状态统计

## 8. 常见问题排查

### Q: Docker 服务启动失败

```bash
# 查看详细日志
docker compose logs <service-name>

# 常见原因：端口冲突
netstat -tlnp | grep 3000  # 检查端口占用

# 重启单个服务
docker compose restart server
```

### Q: 数据库连接失败

```bash
# 检查 MySQL 容器状态
docker compose ps mysql

# 进入 MySQL 容器
docker compose exec mysql mysql -uroot -p

# 检查数据库是否存在
SHOW DATABASES;
```

### Q: 视频加密失败

- 确认 FFmpeg 已安装：`docker compose exec server ffmpeg -version`
- 检查 MinIO bucket 是否创建：访问 http://your-server:9001
- 检查 MASTER_KEY 是否正确配置

### Q: 客户端无法同步视频

- 检查客户端网络连接
- 确认设备已绑定门店（管理后台查看设备列表）
- 检查活动状态是否为「进行中」
- 查看客户端日志

### Q: EMQX 设备连接失败

- 检查 EMQX 服务状态：`docker compose ps emqx`
- 访问 EMQX Dashboard: http://your-server:18083 (默认: admin/public)
- 检查 mqtt_user 表中设备凭证

### Q: 管理后台无法访问

- 检查 admin 容器状态：`docker compose logs admin`
- 确认端口 5173 未被防火墙拦截
- 检查 Nginx/反向代理配置
