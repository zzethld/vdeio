业务背景与目标

🎯 核心场景

  • 连锁品牌 1000+ 门店的培训/宣传视频分发
  • 每天约 10GB 新视频，几 GB 旧视频过期
  • 营销活动驱动：活动关联视频 + 分配门店 + 时间段
  • 同一活动下所有门店看到相同视频列表

⚙️ 关键约束

  • 门店设备：普通 Windows PC（4-8GB RAM）
  • 店员登录：钉钉扫码（个人账号）
  • 按需播放，非数字标牌循环
  • 离线播放：密钥本地缓存 7 天
  • 加密：AES-128 HLS（MVP），可升级 ClearKey DRM

# FFmpeg 加密命令（server/src/services/encryption.ts）
ffmpeg -i input.mp4 \
  -hls_time 10 \
  -hls_key_info_file key_info \
  -hls_playlist_type vod \
  -hls_segment_filename "seg_%03d.ts" \
  output.m3u8

MASTER_KEY（32 字节 hex）存于环境变量，不入库。更换后存量视频无法解密。

密钥分发与离线播放

sequenceDiagram participant P as Shaka Player participant E as Electron 主进程 participant SS as safeStorage participant S as Server API P->>P: 读取 m3u8 → 发现 EXT-X-KEY URI P->>E: 请求密钥 URL (含 JWT + deviceId) E->>SS: 查本地缓存 (7天 TTL) alt 缓存有效 SS-->>E: 返回缓存的 16 字节密钥 else 缓存过期/不存在 E->>S: HTTPS 请求密钥 S->>S: 验证 JWT + 设备权限 + 时间段 S-->>E: 解密后的 AES key E->>SS: 存入 safeStorage (OS 级加密) end E-->>P: 16 字节二进制密钥 P->>P: 解密 ts 分片 → 播放

🔒 安全特性

  • 预签名 URL 绑定 deviceId，24h 过期
  • 密钥 AES-256 加密存 MySQL
  • safeStorage 绑定 OS 用户 + 机器
  • JWT 签名 + 请求频率限制

📴 离线策略

  • 密钥缓存 7 天（断网可播）
  • 播放日志/遥测本地队列（48h）
  • 联网后批量上报
  • 磁盘 >85% 暂停下载，>95% LRU 淘汰

// client/electron/sync-service.ts — 核心同步逻辑
async function syncContent() {
  const cachedIds = readLocalManifest();      // 本地已有视频 ID
  const diff = await api.post('/devices/me/sync', { cachedVideoIds: cachedIds });
  await Promise.all([
    ...diff.downloads.map(v => downloadWithResume(v)), // HTTP Range 断点续传
    ...diff.deletes.map(v => evictLocal(v)),           // LRU 淘汰
  ]);
  await writeLocalManifest();                // 更新 .sync_state.json
}
