# 连锁门店视频管理系统 — 技术说明

## 系统简介

一套面向 1000+ 连锁门店的视频管理与分发系统。总部管理员按"营销活动"统一管理视频内容，门店店员通过钉钉扫码登录桌面客户端按需播放。视频经 AES-128 加密后分发到门店本地缓存，支持离线播放和远程设备管理。

## 技术点

### 1. 视频加密：AES-128 HLS

- 上传 MP4 → FFmpeg 切片为 HLS（m3u8 + ts）+ AES-128 加密
- 每个 ts 分片 10 秒，每个视频独立密钥
- 密钥 AES-256 加密后存 MySQL，master_key 仅存环境变量
- 播放时：客户端请求密钥 URL → 服务端验证权限 → 返回密钥 → 本地解密播放
- 离线时：密钥缓存到 Electron safeStorage（OS 级加密），7 天有效

```
技术栈：FFmpeg 6.x | crypto (Node.js) | Electron safeStorage
```

### 2. 视频分发：HTTP 边缘缓存

- 云端 MinIO 存全量视频，门店本地磁盘做边缘缓存
- ts 分片通过认证端点下载（`/api/v1/videos/:id/segment/:seq`），每次验证 JWT + 设备权限
- 播放优先本地缓存，无缓存则在线播放 + 后台静默下载
- 同步服务作为 Electron Worker Thread 后台运行

```
技术栈：MinIO | HTTP Range（断点续传）| Electron Worker Thread
```

### 3. 增量同步：客户端上报清单 + 服务端计算差异

- 客户端扫描本地已缓存视频 ID → POST 到服务端
- 服务端查询该门店当前应缓存的视频集合（由活动关联计算）→ 做 Set 差集
- 返回 `{ downloads: [...], deletes: [...] }`
- 触发时机：应用启动、MQTT 推送、定时检查

```
技术栈：Set 差集计算 | MQTT 推送 | gzip 压缩传输
```

### 4. 营销活动模型

- 核心数据模型：`活动 → 视频（M:N）→ 门店（M:N）`
- 活动有状态机：`draft → active → ended`
- 活动到期 → 定时任务自动标记 ended → MQTT 推送 → 门店自动删除相关视频
- 门店看到的视频 = 所有进行中活动中关联的视频合集

```
技术栈：MySQL 多表 JOIN | 定时任务 | 活动状态机
```

### 5. 设备通信：EMQX MQTT

- 设备 MQTT 连接认证（deviceId + bcrypt token）
- 遥测数据每 5 分钟上报（CPU/内存/磁盘/缓存大小）
- 远程命令：重启、强制同步、清理缓存
- 在线状态：MQTT Will 遗嘱消息，断开自动标记离线
- 断网保护：遥测和播放日志本地缓存 48h，联网后批量上报

```
技术栈：EMQX 5.8 | MQTT.js | QoS 1 | Will 遗嘱
```

### 6. 登录与设备绑定：钉钉 OAuth + 自动绑定

- 钉钉企业内部应用扫码登录
- 首次扫码：JWT payload 含 storeId → 客户端自动将设备绑定到该门店
- 后续登录：设备已有 storeId，直接获取该门店视频列表
- 双 token：accessToken（2h）+ refreshToken（7d）

```
技术栈：钉钉开放平台 SDK | JWT (HS512) | Electron safeStorage
```

### 7. 客户端安全

- Electron 上下文隔离 + 沙箱 + CSP 策略
- 禁用 DevTools、防截图（setContentProtection）
- 生产环境反调试检测
- 密钥通过 protocol.handle 拦截，不经过 JavaScript 网络层

```
技术栈：Electron 32 | contextIsolation | protocol.handle
```

### 8. 磁盘空间管理

- 磁盘 >85%：暂停下载 + 上报告警
- 磁盘 >95%：LRU 淘汰最久未播放的视频，释放到 80%
- 管理后台显示各门店磁盘使用率

## 技术栈汇总

| 层级 | 技术 |
|------|------|
| 后端 | Node.js 18 + Express 4 + TypeScript 5 |

| 缓存 | Redis 7 |
| 存储 | MinIO |
| 设备通信 | EMQX 5.8 |

## 核心数据流

```
上传视频 → FFmpeg 加密 → MinIO 存储
                              ↓
        创建活动 → 关联视频 → 分配门店
                              ↓
        MQTT 推送通知 → 门店自动下载
                              ↓
        店员扫码 → 按活动浏览 → 点击播放
                              ↓
        本地缓存命中 → 直接播放（零延迟）
        本地无缓存 → 在线播放 + 后台下载
                              ↓
        活动到期 → 自动标记 ended → 门店自动清理
```
